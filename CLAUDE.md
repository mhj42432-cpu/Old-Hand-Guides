# 智能体项目总文件夹

## 语言规则

- 所有交互使用中文，包括代码注释、文档、对话等。
- 代码中的变量名、函数名可使用英文，但注释和说明用中文。

## 项目规范

### 文件组织
- 每个子项目放在独立的子文件夹中。
- 公共资源放在 `shared/` 目录下。
- 每个子项目应包含自己的 README.md 说明文件。

### 代码风格
- 优先使用简洁、可读性高的代码。
- 不过度抽象，三个相似的行好过一个过早的抽象。
- 不写无意义的注释，只注释"为什么"而不是"是什么"。
- 不在代码中引用任务编号、PR 编号等会过时的信息。

### 版本控制
- 使用 Git 进行版本管理。
- 提交信息使用中文，格式：`类型: 简要描述`（如 `功能: 添加用户登录`, `修复: 修正分页错误`）。
- 不要自动提交，除非用户明确要求。

### 安全
- 敏感信息（密钥、密码等）不提交到仓库，使用环境变量或 .env 文件。
- .env 文件必须加入 .gitignore。

### 依赖管理
- 各子项目独立管理依赖。
- 定期检查依赖更新，但不要自动升级。
